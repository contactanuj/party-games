## Architecture — reuse the shared word engine (don't fork it)

- Build on `@partydeck/core/word-engine` (`window.WordCore`). One pure, deterministic engine powers
  every game in this family **by configuration**, not by copying:
  - `contentModel`: `word` (everyone shares a word; outsider has none) · `wordPair` (outsider gets a
    close, different word) · `locationRoles` (everyone shares a location + a personal role; outsider
    has neither).
  - `interaction`: `clues` (each says one word in order) · `questions` (the app poses questions,
    answered in turn) · `play` (free-form Q&A on a timer; accuse/guess can interrupt).
  - `guessMode`: `free` · `list` · `fromTopic` · `none`; plus scoring "buckets", voting modes,
    tie-breakers, and `caughtCanGuess` / `allowOutsiderEarlyGuess`.
- Author only: `apps/<game>/src/content.js` (`window.WORD_CONTENT` — packs of data, with optional
  per-item `c:[...]` bot clue hints), `src/game.js` (`window.WORD_GAME` — the engine def +
  `meta` with title/tagline/variant presets/help/`optionLabels`/`hiddenOptions`), `src/theme.css`,
  `identity.json`, `App.js`, `build.js`, `scripts/make-icon.js`.
- The Expo shell loads one inlined `assets/app.html` produced by `build.js` →
  `composeWordAppHtml` (inlines base+theme CSS, `word-engine.js`, `content.js`, `game.js`,
  `sound.js`, `word-ui.js`). Guard against a literal `</script>`. `app.json` is generated by
  `make-app-json.js`; icons/splash by `make-icon.js`. **Never hand-edit generated output.**
- Engine is ES5/UMD, **no DOM, no network**, JSON-serializable state, a `phase` state machine,
  throwing actions, and a **seeded PRNG on `state.rngState`** so a whole match (humans + bots)
  replays from `(config, seed, inputs)` byte-for-byte.

## Configurability — maximum, but guarded (never confusing or breakable)

- `defaultConfig(playerCount)` returns a faithful, start-ready config; **every practical variation
  is a field** on it. Offer friendly **variant presets** in `meta.variants` (e.g. easy/standard/hard)
  that patch a bundle of fields, with all options still available in Advanced settings.
- `validateConfig(config, library)` returns `{ok, errors, warnings}`:
  - **errors block starting** an illegal / unwinnable / nonsensical game (e.g. outsiders ≥ half the
    table, no content selected, blank names);
  - **warnings** allow off-spec-but-playable setups.
- The UI must **disable the proceed/Done button while invalid** and show validation **at the top** —
  never let a player leave a screen with an unstartable config, and never rely on a buried error.
- **Steppers/controls must respect the player count** (e.g. the outsider-count max is derived from
  players) so most invalid states cannot be entered at all.
- Retest with this intuition everywhere: walk every flow and make it easy, intuitive, and
  practically intelligent.

## Secrecy / information boundary — the #1 rule (the wink-killer lesson)

A shared/observable screen revealing hidden info ruins the game. In Wink Killer we shipped a bug
where role-coded **icons and colours leaked roles on the general (all-visible) screen and on the
re-check list** — never again.

- Every shared surface (lobby, turn order, clue recap, vote list, timer, scoreboard) renders **only**
  from `publicState()` and shows **no** role / secret word / location / team, and **no role-coded
  colour, icon, ordering, or emphasis**. The **theme must not leak hidden data anywhere**.
- The secret is shown **only** behind the per-player handoff **gate**, to one player who confirmed
  it's them; the card is **shape-identical for every role** (same colours/layout/buttons — only the
  words differ; e.g. in a `wordPair` game nobody is even told they're the odd one out).
- Roles are **timer-based**: the secret card shows a short **auto-hide countdown** (wink-killer
  style) so a role is never left on screen — and the player can always **re-check** their secret via
  the **same gated single-player handoff** (never a list of everyone).
- The full truth appears only **after** the round ends. Add UI smoke tests that assert the secret
  never appears on any shared screen, that the reveal/recheck are gated, and (for `wordPair`) that
  every reveal card is identical.

## Bots (offline, only if practical)

- Support computer players the same way as the rest of the suite: the last `config.botCount` seats
  are bots; their choices come from the engine PRNG (deterministic) and they use **only their own
  private knowledge** (never anything a human couldn't deduce).
- These are **talking** games: be honest that bots can fill seats + vote but cannot truly bluff —
  surface that caveat in the lobby (`def.botsImpractical`). Don't pretend otherwise.

## View / platform correctness

- **No status-bar overlap.** `base.css` pads `env(safe-area-inset-top/right/left/bottom)`; generated
  `app.json` uses a **non-translucent Android status bar**; the shell sets `translucent={false}`.
- **Keyboard.** Android `softwareKeyboardLayoutMode: resize` and scroll-to-top on every render, so
  **focusing any input never pushes the UI up under the status bar**.

## Icon

- Generate a **clean, professional, practical, inviting** 1024×1024 icon (and a splash) with the
  dependency-free `packages/core/build/png-canvas.js` SDF helper — a simple, legible emblem that
  reads at a glance and matches the brand theme. No role-coded meaning.

## Tests & tooling

- Add `apps/<game>/tests/engine.<game>.test.js` (config/validation, content sanity, the win/scoring
  matrix, determinism + JSON round-trip, a fuzz loop that never throws and never leaks) and
  `tests/ui.<game>.smoke.test.js` (drive the real UI via the node DOM stub; assert the no-leak,
  gated-reveal, timer-reveal, and disable-on-invalid behaviours).
- `node scripts/run-tests.js` (all tests, dependency-free) must pass.

## Repo / open-source hygiene

- Keep the open-source files current (README, NOTICE with the originality/trademark note — use a
  **generic name** for any trademarked game, like Spyfall → "Spy Hunt"), CHANGELOG, CONTRIBUTING,
  CODE_OF_CONDUCT, SECURITY, issue/PR templates, and CI.
- **Never commit secrets** (Expo tokens, keystores). CI reads `EXPO_TOKEN` from a GitHub Actions
  secret; rotate any token shared in plaintext.
- Commit in focused commits with clear messages. **Do not add a Claude/AI co-author trailer.**
- Don't bundle unrelated in-progress changes from other games; stage only this game's files.
